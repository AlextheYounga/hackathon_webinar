# Introductory AI Tools

## Goal

This section is for students who are new to AI development tools.

The main idea:

> You do not need to be an expert programmer to use AI tools, but you do need to understand what kind of tool you are using.

---

# Tool Categories

## 1. Chat Assistants

Examples:

- ChatGPT
- Claude
- Gemini

Best for:

- Explaining concepts
- Brainstorming project ideas
- Writing project plans
- Debugging error messages
- Generating small code snippets
- Creating documentation
- Reviewing your idea

Good prompt:

```md
I am building a hackathon project.

The idea is:

[describe idea]

Please help me turn this into:

1. A simple one-day build plan
2. The smallest possible feature list
3. A demo script
4. Technical risks
````

## 2. Browser-Based App Builders

Examples:

* Lovable
* Wix
* Bolt
* Replit Agent
* v0
* Base44
	* It can help with:
		- App structure
		- User interface
		- Database-like features
		- Authentication-style flows
		- Hosting
		- Fast prototyping

	* Good For
		- Product demos
		- Internal tools
		- Simple apps
		- MVP-style prototypes
		- Projects where speed matters more than engineering control

These tools let you describe an app in natural language and generate a working project.

Best for:

* Beginners
* Fast prototypes
* Landing pages
* Dashboards
* Simple full-stack apps
* Visual demos
* Product mockups

## Why Browser-Based Tools Are Useful

They reduce setup problems.

You may not need to install:

* Node.js
* Python
* Databases
* Git
* Docker
* Local editors
* Package managers

That is a big advantage during a hackathon.

## Common Weaknesses

Browser-based tools can still create problems:

* The app may look good but not really work
* The database design may be weak
* Authentication can become confusing
* The generated code may be hard to understand
* The project may become difficult to customize
* You may hit platform limits
* You may not know how to fix bugs when the AI gets stuck

## Recommended Beginner Workflow

1. Start with a clear idea
2. Ask the AI to make a plan before building
3. Build the smallest possible version
4. Test every feature manually
5. Stop adding features early
6. Prepare the demo

# Beginner Advice

Use AI to move faster, but keep the project simple.

A simple working demo beats an ambitious broken app.
